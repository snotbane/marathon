# Quickstart

1. Install this addon to `res://addons/sunkist`

2. Enable the plugins in __Project Settings__:
	- `Sunkist`
	- `Sunkist :: Task Runner`

3. Open the new `Tasks` plugin window (in the header bar).

4. Click the `Settings` button to open the Task Runner settings in the Inspector.

5. In the Inspector, click the `Install Python Venv` button. This will install a `python3` virtual environment inside the addon folder.
	- Consider adding `.venv` to your project's `.gitignore`.

6. Take a look at the [Task Runner](/.docs/TaskRunner.md) documentation to get started!

# Sunkist Plugin

The base Sunkist plugin provides an importer for `SunkistSheet` (`.sun`) resources. Opening one of these will refresh its subassets.

Additionally, it provides a new main screen panel with which the user can preview `SunkistTexture`s from the FileSystem.
- The left panel (green) will display the `AtlasTexture`s which compose the `SunkistTexture`.
- The right panel (blue) will display a 3D render of this `SunkistTexture`.

![Tutorial02](/.docs/sunkist_tutorial_02.png)

For more information about how to create these assets, see the [__Sunkist Assembler Task__](/.docs/SunkistAssemblyTask.md) documenation.


## Resource Types

Several new Resources are in use.

### `SunkistSheet`

This is a file generated by the [__Sunkist Assembler Task__](/.docs/SunkistAssemblyTask.md) and contains all data to create `SunkistAtlasTexture`s and `SunkistTexture`s.

### `SunkistAtlasTexture`

This is a simple extension to `AtlasTexture`, which also provides the offset position of the bounding box (in the original image, without needing all the extra transparent space).

### `SunkistTexture`

This is the main type of texture to be utilized. It contains an array of textures, which are automatically assigned by `SunkistSheet` upon refresh.

### `SunkistMaterial`

A self-managing material which, when assigned a `SunkistTexture`, will pass the proper textures to its shader.


## Nodes

In a 3D scene, there are a few new `Node3D`s to use to quickly combine different sprite layers together to create a fully rendered 3D sprite.

![Tutorial03](/.docs/sunkist_tutorial_03.png)

> [!IMPORTANT]
> The X and Y positions of all these Nodes are automatically managed. It is assumed they will always face the relative positive Z axis. If you need to modify their orientation, consider adding a parent `Node3D` or changing [`SpriteBase3D.offset`]() to offset their transform.
>
> Sprite order is controlled by setting each `Node3D.position.z`. To ensure layer order are maintained, make sure to check `Keep Camera Depth` on all desired elements.

### `Sunkist3D`

Like [`Sprite3D`](), but uses a `SunkistMaterial` to properly display its assigned `SunkistTexture`.

### `AnimatedSunkist3D`

Like [`AnimatedSprite3D`](), but uses a `SunkistMaterial` to properly display its assigned `SunkistTexture`.

### `SunkistContainer3D`

This should be the ancestor of any `Sunkist3D` or `AnimatedSunkist3D` nodes. Make sure that you set the `size` to the (uniform) size of the source images.
